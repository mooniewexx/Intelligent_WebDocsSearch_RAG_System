# Intelligent Web Documentation Search RAG System

Этот репозиторий содержит ноутбук с реализацией и экспериментальной оценкой интеллектуальной системы поиска информации в корпоративной веб-документации на основе RAG-подхода.

Проект подготовлен в рамках ВКР на тему: **«Разработка интеллектуальной системы поиска информации в корпоративной документации на основе RAG подхода»**.

## Описание проекта

Цель проекта — разработать и проверить пайплайн, который позволяет находить релевантные фрагменты корпоративной документации и формировать по ним ответ на естественном языке.

В ноутбуке реализованы следующие этапы:

1. загрузка и извлечение данных из веб-документации;
2. загрузка локальных файлов, архивов и внешних ссылок;
3. очистка и нормализация текстовых данных;
4. разбиение документов на смысловые фрагменты;
5. построение гибридного индекса;
6. поиск релевантных фрагментов по пользовательскому запросу;
7. реранжирование найденных кандидатов;
8. фильтрация фрагментов через LLM-grader;
9. генерация итогового ответа на основе найденного контекста;
10. сравнение RAG-подхода с baseline-режимами;
11. оценка качества ответов по автоматическим метрикам и LLM-as-a-judge;
12. визуализация результатов экспериментов.

## Используемый подход

Система построена по архитектуре Retrieval-Augmented Generation.

Общая схема работы:

```text
Документация
    ↓
Загрузка и очистка текста
    ↓
Разбиение на чанки
    ↓
Построение гибридного индекса
    ↓
Поиск кандидатов по запросу
    ↓
Реранжирование
    ↓
Фильтрация через LLM-grader
    ↓
Генерация ответа
    ↓
Оценка качества
```

Для поиска используются два типа представления текста:

- **dense-поиск** — семантический поиск по embedding-векторам;
- **sparse-поиск** — лексический поиск по BM25;
- **hybrid retrieval** — объединение dense- и sparse-оценок.

После первичного поиска найденные фрагменты дополнительно пересортировываются reranker-моделью и фильтруются через LLM-grader. Это позволяет передавать в генератор не просто похожие по теме фрагменты, а те, которые действительно содержат информацию для ответа.

## Сравниваемые режимы

В ноутбуке сравниваются несколько режимов генерации ответа:

| Режим | Описание |
|---|---|
| `LLM_no_context` | модель отвечает без внешнего контекста |
| `LLM_full_context_truncated` | весь корпус объединяется в один контекст и обрезается по лимиту токенов |
| `LLM_full_context_controlled` | контекстное окно заполняется документами последовательно до заданного лимита |
| `RAG` | система ищет релевантные фрагменты, реранжирует их и генерирует ответ по найденному контексту |

Такое сравнение позволяет проверить, насколько использование внешнего поиска и отбора фрагментов улучшает качество ответов по сравнению с прямой генерацией.

## Метрики качества

Для оценки ответов используются:

- Exact Match;
- Token F1;
- ROUGE-L F1;
- semantic cosine similarity;
- LLM-as-a-judge;
- correctness;
- completeness;
- groundedness;
- latency;
- error rate;
- empty prediction rate.

Результаты сохраняются в виде CSV-файлов с детальными предсказаниями и JSON-файлов с агрегированными метриками.

## Как запустить ноутбук

1. Склонируйте репозиторий или скачайте ноутбук:

```bash
git clone https://github.com/<username>/<repository-name>.git
cd <repository-name>
```

2. Откройте ноутбук:

```bash
jupyter notebook Intelligent_WebDocsSearch_RAG_System.ipynb
```

или загрузите его в Google Colab.

3. Выполните ячейки с установкой зависимостей.

4. Укажите необходимые параметры в конфигурационном блоке ноутбука:

```python
BASE_URL = "..."
INDEX_DIR = Path("...")
BENCHMARK_PATH = "..."
LLM_MODEL_NAME = "..."
EMBED_MODEL_NAME = "..."
RERANKER_MODEL_NAME = "..."
```

5. Настройте доступ к LLM/embedding/reranker API. Не публикуйте API-ключи в открытом репозитории. Лучше использовать переменные окружения или секреты Colab.

6. Запустите ноутбук последовательно по разделам:

```text
1. Подготовка окружения
2. Загрузка документов
3. Чанкинг
4. Построение RAG-пайплайна
5. Проверка системы
6. Baseline-подходы
7. Бенчмарк
8. Итоговое сравнение
```

## Важное замечание о воспроизводимости

Ноутбук рассчитан на работу с внешними моделями и конкретным корпусом документации. Поэтому для полного воспроизведения экспериментов необходимо:

- иметь доступ к используемому API моделей;
- указать актуальный адрес документации;
- подготовить файл с контрольными вопросами и эталонными ответами;
- проверить пути к директориям индекса и результатам.

Если доступ к исходной корпоративной документации или API отсутствует, ноутбук можно использовать как шаблон для запуска RAG-пайплайна на другом наборе документов.
